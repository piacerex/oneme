# oneme ROADMAP

Ready Player Me のように、Webで作成した3Dアバターを保存し、外部アプリへ
配信できるサービスを作る。

今回の最優先体験は次の縦切りである。

1. Web UIでアバターを選択・編集する
2. 顔写真を同意付きで読み込む
3. 顔の輪郭に沿って一時的に切り出す
4. 切り出した顔写真をアバターの顔面へテクスチャとしてマッピングする
5. 疑似3Dパラメータを反映したアバターを回転プレビューする
6. アバター設定を保存し、GLBまたはFBXとして書き出す

## 開発方針

- バックエンドは Elixir / Phoenix を使用する。
- 画面は Phoenix LiveView を基本とし、Three.js の描画とファイル処理は JavaScript Hook に分離する。
- PostgreSQL は保存用、S3互換ストレージは公開アセットとエクスポート成果物用に使う。
- ランタイム配信は `.glb` を第一形式とする。
- `.fbx` はブラウザ内で直接生成せず、Blenderを使うサーバー側のエクスポートジョブで作る。
- `.vrm` はGLBの実出力とリグ検証ができた後に追加する。
- 顔写真のマッピングは顔テクスチャの一時表示から始め、完全な3D顔復元や本人そっくりのメッシュ生成とは分けて扱う。
- 元写真と、そこから作ったエクスポート用の派生テクスチャは別データとして管理する。
- 元写真は原則保存せず、派生テクスチャをモデルへ埋め込む場合は別途明示同意、削除、保持時間を管理する。
- 最初は少数のライセンス確認済みアセットで、編集から保存までの操作感を完成させる。

## Phase 2: 顔写真マッピング縦切り

### 機能

- [x] 写真利用の同意チェック
- [x] 顔写真アップロード
- [x] `FaceDetector`を優先した顔検出と輪郭に沿った一時的な切り出し
- [x] 切り出した写真の顔面テクスチャマッピング
- [x] マッピングのオン・オフとクリア
- [x] 肌色、髪色の自動推定
- [x] 顔の縦横比と奥行きを疑似3Dパラメータへ変換
- [x] 推薦結果と顔比率を手動編集して通常の保存フローへ戻す
- [x] エクスポート時に顔写真由来の派生テクスチャを含めるか選択する

### 実装方針

- 最初の切り出しはブラウザ側で行い、元写真をサーバーへ送らずに成立させる。
- 顔検出は標準APIまたは実績のあるブラウザ向けライブラリを使い、輪郭マスクを手作業の矩形切り出しにしない。
- `FaceDetector`未対応ブラウザでは中央基準の安全なフォールバックを使い、画面上で検出結果と区別して表示する。
- Three.js Hookは、顔テクスチャ、透明度、顔のスケール、奥行きオフセットを受け取って描画する。
- 顔テクスチャはセッション中だけ保持し、保存時には元写真を設定JSONへ含めない。
- エクスポート用テクスチャを作る場合は、元写真から輪郭マスク済みの派生テクスチャを生成し、元写真と分離する。
- 派生テクスチャをGLB/FBXへ含める前に、ユーザーへ「顔写真由来のテクスチャをモデルへ埋め込む」ことを示して同意を取得する。
- エクスポートしない場合は、顔写真マッピングをプレビュー専用として破棄できる。
- サーバー解析が必要になった場合は、Phoenixの一時ジョブと自動削除を追加する。

## 最初のマイルストーン

1. Phoenixアプリを起動し、LiveViewのビルダー画面を表示する
2. Three.jsで実アセットのベースアバターを表示する
3. パーツ選択と保存・復元をPostgreSQLへ接続する
4. 顔写真をブラウザ内で輪郭切り出しする
5. 切り出した顔写真を3D顔面へマッピングする
6. アバターをゆっくり回転させながらマッピングを確認する
7. 疑似3Dパラメータを保存・復元する
8. 実メッシュGLBを生成する
9. Blender経由でFBXを生成する
10. 公開URLとモデルURLを発行する

## リスクと判断

- 完全な3D顔復元は研究開発として別トラックに分離する
- 顔写真マッピングはUV展開された顔メッシュが前提になる
- アセットごとにUV、原点、スケール、ボーンが違うと合成できない
- FBXはソフトウェアごとの軸・マテリアル差が大きいため、GLBを正規形式にする
- 顔写真と顔写真由来の派生テクスチャは生体情報に近いデータとして、同意、削除、保持時間、ログを設計する
- 派生テクスチャを含むエクスポート成果物は、元写真を削除しても独立して残り得るため、同時削除方針を設計する
- 実アセットが揃わない場合は、プレースホルダーのまま完了扱いにしない

## 完了判定

各Phaseの完了は、画面が存在することだけではなく、次の3つで判定する。

1. ユーザー操作を通したブラウザSmoke Test
2. APIレスポンスと保存データのスキーマ検証
3. 実アセットまたは明示したテストアセットによる読み込み検証

ローカルMVPや契約テストの完了と、本番運用可能な実装の完了は分けて記録する。
